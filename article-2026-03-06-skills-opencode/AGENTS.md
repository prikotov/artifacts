# AGENTS.md — Гайд по исследованиям и статьям для блога prikotov.pro

## О проекте

**Блог:** prikotov.pro — технический блог о AI, разработке, автоматизации

**Цель блога:** Неявное формирование интереса к проекту [task.ai-aid.pro](https://task.ai-aid.pro)

**Тон:** Честный, практичный, от первого лица. Не "эксперт", а практик, который делится опытом.

**Доступные skills:**
- `google-trends` — тренды поиска
- `yandex-metrika-search` — поисковые фразы из Метрики
- `yandex-metrika-traffic` — источники трафика
- `yandex-metrika-pages` — статистика страниц
- `yandex-metrika-visitors` — анализ посетителей
- `yandex-webmaster-queries` — запросы из Вебмастера

## Процесс исследования и написания статьи

### Этап 1: Планирование сбора данных

**Цель:** Понять какие данные нужны и откуда их получить.

**Шаги:**

1. **Определить тему статьи**
   - Что хочет рассказать автор?
   - Какая проблема решается?
   - Как это связано с task.ai-aid.pro?

2. **Составить список вопросов**
   - Что нужно узнать о трендах? → Google Trends
   - Что нужно узнать о текущем трафике? → Яндекс.Метрика
   - Что нужно узнать о поисковых запросах? → Яндекс.Вебмастер
   - Какие ключевые слова в тренде?

3. **Выбрать skills для использования**
   ```
   | Вопрос | Skill | Ожидаемый результат |
   |--------|-------|---------------------|
   | ... | ... | ... |
   ```

4. **Зафиксировать план в файл**
   - Создать папку: `research/YYYY-MM-DD - [тема исследования]/`
   - Создать файл: `01 - План исследования.md`
   - Записать гипотезы, вопросы, skills

**Выход:** `research/YYYY-MM-DD - [тема]/01 - План исследования.md`

---

### Этап 2: Сбор данных

**Цель:** Собрать сырые данные из всех источников.

**Порядок:**

1. **Google Trends — сначала**
   ```
   /skill google-trends
   ```
   - Анализ ключевых слов по теме
   - Сравнение нескольких запросов
   - География: RU (или worldwide для глобальных тем)
   - Период: последние 3-12 месяцев

   Примеры запросов:
   ```
   > Сравни тренды: "ai skills", "opencode", "cursor ai"
   > Покажи рост интереса к "glm 5" за последний год
   ```

2. **Яндекс.Метрика — статистика блога**
   ```
   /skill yandex-metrika-pages
   /skill yandex-metrika-traffic
   /skill yandex-metrika-search
   /skill yandex-metrika-visitors
   ```
   - Топ страниц за период
   - Источники трафика
   - Поисковые фразы
   - Демография посетителей

3. **Яндекс.Вебмастер — поисковая выдача**
   ```
   /skill yandex-webmaster-queries
   ```
   - Запросы с показами
   - Позиции в выдаче
   - CTR по запросам
   - Возможности для улучшения

**Важно:** 
- Сохранять отчёты с датой
- Не интерпретировать на этом этапе — только собирать

**Выход:** Папка с отчётами `YYYY-MM-DD/`

### Этап 3: Обработка данных и выявление инсайтов

**Цель:** Превратить сырые данные в выводы, применимые к блогу.

**Шаги:**

1. **Анализ трендов**
   - Какие темы растут?
   - Какие падают?
   - Есть ли сезонность?
   - Что сейчас "на хайпе"?

2. **Анализ текущего блога**
   - Какие статьи заходят? Почему?
   - Какие не заходят? Почему?
   - Где низкий CTR — почему?
   - Какие запросы упускаются?

3. **Поиск связей**
   - Тренды vs текущий контент — где разрыв?
   - Что ищут, но у меня нет?
   - Что у меня есть, но плохо оптимизировано?

4. **Инсайты для task.ai-aid.pro**
   - Какие темы привлекают целевую аудиторию?
   - Как мягко упомянуть проект?
   - Какой контент показывает экспертизу в AI-агентах?

**Формат вывода:**
```markdown

### Тренды
- ...

- ...

### Возможности
- ...

### Связь с task.ai-aid.pro
- ...
```

### Этап 4: Стратегия статьи

**Цель:** Определить как статья достигнет целей.

**Вопросы:**

1. **Целевая аудитория**
   - Кто будет читать?
   - Какая у них проблема?
   - Что они ищут?

2. **Ключевые слова**
   - Основное KW (в заголовке)
   - Вторичные KW (в тексте)
   - Трендовые KW (для хайпа)

3. **Позиционирование**
   - Уникальность: чем статья отличается от других?
   - Ценность: что читатель получит?
   - CTA: что читатель должен сделать?

4. **Связь с task.ai-aid.pro**
   - Контекст: где упомянуть естественно?
   - Ценность: как показать экспертизу?
   - Мягкость: не продаём, а делимся

5. **Формат**
   - Туториал / кейс / обзор / мнение
   - Длина: короткая / средняя / длинная
   - Элементы: код, диаграммы, скриншоты

**Выход:** Файл `research/YYYY-MM-DD - [тема]/03 - Стратегия статьи.md`

### Этап 6: Драфт статьи

**Цель:** Написать первый черновик.

**Принципы:**

1. **Голос автора**
   - "У меня нет опыта в..." — честность
   - "Я решил попробовать..." — действие
   - "Оказалось, что..." — открытие
   - Не "эксперт", а практик

2. **Структура**
   - Следовать плану
   - Короткие абзацы (2-4 предложения)
   - Подзаголовки каждые 300-500 слов

3. **Практичность**
   - Код, который можно запустить
   - Команды, которые работают
   - Примеры из реального опыта

4. **Мета-элементы**
   - Диаграммы mermaid (кириллица в кавычках)
   - Плейсхолдеры для изображений с промптами
   - Ссылки на GitHub, блог, task.ai-aid.pro

5. **SEO**
   - Ключевое слово в заголовке
   - Ключевые слова в H2/H3
   - Естественное вхождение в текст

**Формат файла:**
```markdown
# [Заголовок с ключевым словом]

<!-- OG-изображение: [промпт для генерации] -->

[Введение — хук, проблема, решение]

## [Секция 1]

<!-- Изображение: [промпт] -->

[Текст]

## [Секция 2]

[Текст + примеры кода]

...

## Итог

[Краткий вывод + CTA]

---

**Ссылки:**
- [GitHub](...)
- [Блог](https://prikotov.pro)
- [task.ai-aid.pro](https://task.ai-aid.pro)
```

**Выход:** Файл `research/YYYY-MM-DD - [тема]/05 - Статья (драфт).md`

## Типичные ошибки

1. **Mermaid с кириллицей** → заключать в кавычки
   - ❌ `x-axis [Янв, Фев, Мар]`
   - ✅ `x-axis ["Янв", "Фев", "Мар"]`
   - ✅ `x-axis [Jan, Feb, Mar]` (английский — без кавычек)

2. **Mermaid timeline** → использовать `section`, не `00:00 :`
   - ❌ `00:00 : Событие`
   - ✅ `section Название\n    Событие : Детали`

3. **Продажа вместо истории** → делиться опытом
   - ❌ "Вам нужен task.ai-aid.pro"
   - ✅ "Мой блог помогает привлекать внимание к task.ai-aid.pro"

4. **Экспертный тон** → честный тон
   - ❌ "Как эксперт, я рекомендую..."
   - ✅ "У меня нет опыта в продвижении. Я попробовал..."

5. **Сбор данных без плана** → сначала план
   - ❌ Сразу запускать все skills
   - ✅ Сначала определить вопросы, потом выбирать skills

6. **Нет связи с целью блога** → всегда помнить про task.ai-aid.pro
   - ❌ Статья ради статьи
   - ✅ Статья, которая показывает экспертизу в AI-агентах

7. **Инсайты без ссылок на данные** → всегда указывать источники
   - ❌ "Тренд растёт"
   - ✅ "Тренд растёт [(данные)](../google_google_trends_reports/...)"

## Быстрый старт для нового исследования

Когда пользователь просит провести исследование или написать статью:

1. **Уточнить тему** — о чём статья?
2. **Создать папку исследования** — `research/YYYY-MM-DD - [тема]/`
3. **Создать файл плана** — `01 - План исследования.md`
4. **Провести сбор данных** — использовать skills
5. **Выявить инсайты** — записать в `02 - Инсайты.md` со ссылками на reports
6. **Сформировать стратегию** — `03 - Стратегия статьи.md`
7. **Написать план статьи** — `04 - План статьи.md`
8. **Создать драфт** — `05 - Статья (драфт).md`
9. **Предложить улучшения** — SEO, визуал, CTA

Готово!
